Associative containers

Conception

The Associative containers implement an associative array which is composed of key+value bindings (pairs) and allows efficiently access the value by the key.
There are two kind of the associative containers

  • Map
  • and Set
    is a sort of associative container in which the stored values themselves are keys.

Multi associative container is the container which allows to store multiple values with the same key. Both the map and set containes have Multi associative container implementation.

And there are two major approaches how to implement the associative array:

  • as a self-balancing binary search tree (Red-Black in particular)
  • and as a bunch of buckets which hold the bindings based on the hash of theirs keys.

There are several implementations of the associative array container:

  • Binary::Search::[RB::]Tree
    An implementation of the simple and self-balancing (Red-Black) Binary Search Tree. RB BST implementation is a base of following containers:
    • [Multi::]Set
      The same as the std::[multi]set
    • [Multi::]Map
      The same as the std::[multi]map
  • Hash::Chaining::Table
    An implementation of the Chaining Hash Table which is a base of following containers:
    • Unordered::[Multi::]Set
      The same as the std::unordered_[multi]set
    • Unordered::[Multi::]Map
      The same as the std::unordered_[multi]map
